§ 02

飞书 → GitHub 替换映射

原方案中飞书承担多个角色：身份认证、任务派发、文档存储、通知中心、CV 公示。 重构后由 GitHub + Supabase + Discord 三件套替代——前者承担技术身份与任务，后者承担实时通知。

飞书 OAuth 登录
→
GitHub OAuth via Supabase Auth
已完成 · 通过 Supabase implicit flow + window.__supabase
飞书企业内组织架构
→
GitHub username + 头像（公开身份）
公开身份替代企业内身份，损失"我是 WebAgentLab 成员"的强绑定
飞书多维表格当数据库
→
Supabase (PostgreSQL) · 4 张表
cv_entries / mail / quest_states / review_tasks 全部完成
飞书任务表双向同步
→
GitHub Issues 双向同步
issue label = workshop · 玩家提交 = comment · 关闭 = 完成
飞书 IM 机器人通知
→
GitHub Notifications + Discord Webhook
Discord 群当 IM 通知中心 · 微信群双通道补救大陆访问
飞书企业内文档
→
GitHub Markdown + GitHub Pages
章程 / 规范 / 工作组介绍 全部以 markdown 形式 PR 维护
飞书共享 CV 公示
→
GitHub Profile + 自建排行榜页
Vercel 公开页拉 Supabase 数据 · 玩家可分享自己的 CV URL
飞书工单（反馈通道）
→
GitHub Issue Template
游戏内反馈按钮 → POST 到 cua-base repo
飞书贡献记录写回
→
GitHub contributor.md PR
CV 通过后系统自动 PR 到 WebAgentLab 组织 repo 的 contributor 表
飞书会议 / 大会
→
GitHub Discussions + 游戏内大集会
Discussions 作提案讨论 · 游戏内大集会广场作仪式现场
双轨思路：以上替换不是非此即彼。如果飞书企业版未来可用， GitHub 路径作为更开放、更社区化的版本可与飞书并行——两条数据流、两种受众。

Phase 1 · 单镇可跑 ✓ 已完成

新人 30 分钟内完成：GitHub 登录 → 进入萌芽镇 → 与阿降对话 → 接到第一个新手任务 → 完成至少 2 个新手任务。 单人模式、不接外部、地图仅萌芽镇。

Phase 2 · 闭环跑通（AI 模拟版）✓ 已完成

任意贡献者能跑完：进入百晓居 → 接任务 → 完成（外部链接） → 回游戏提交 → 3 位 AI 审核员（周明 / 严之 / 白徽）投票 → 收邮件 → CV 入账 → 数据写入 Supabase。

说明：当前是模拟闭环——审核员是 AI、任务是预定义。 真闭环需要 Phase 2.5（GitHub Issues 集成 + 真审核员）。

Phase 2.5 · 真任务源 — 未开始

任务列表从 GitHub Issues 实时拉取，提交时附带 PR/Issue 链接， 验收时关联到真实贡献。至少 1 个 WebAgentLab 真实 repo 的 issue 能在游戏内被接收/提交。

这是从"AI 模拟"切到"真任务"的核心。当前优先级最高。

Phase 3 · 多人在场 ✓ 已完成（轻量版）

Supabase Realtime 替代 Colyseus，14 个 scene 全部接入实时同步。 实现：玩家位置同步 / 实时聊天（世界 + 私聊）/ 好友 / 关注 / 在线状态 / 表情 / E 键互动。 多人系统已在 cua-base.vercel.app 跑通。

说明：当前是轻量版多人（无 Colyseus 房间制 + 无反作弊）。 压测 + 防作弊留给真用户量上来后再做。

Phase 4 · 治理上线 ⋯ 进行中

已完成：议政高地外景 + 三建筑（开元楼/明镜阁/远见塔）+ 提案投票系统 + 申诉流程。 待完成：基于 GitHub Discussions 的真治理流程 / 章程 markdown 渲染。

§ 08

替换决策依据

为什么选择 GitHub 而不是 Discord / 微信群作为飞书的主要替代？

维度飞书GitHubDiscord微信群
任务管理多维表格Issues ★无无
贡献追踪文档Commits / PR / Profile ★无无
真实性企业认证username（公开）★半匿名实名
API 成熟度好但有审批极成熟 ★好几乎没有
大陆免 VPN是否（不稳定）否是
社区匹配度AI 社区已用AI 开发者天然在用 ★通用通用

GitHub 替换飞书技术上比飞书更成熟——但损失了"中国大陆免 VPN 访问"。

补救方案：把 Discord（海外用户）+ 微信群（大陆用户）都接 Webhook， 让通知双通道送达。这是 J9-new 的扩展形态。
